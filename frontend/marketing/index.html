Bulk Import System
Efficient CSV-based question management
üì• How Bulk Import Works
1

Prepare CSV Data

Format questions in CSV: Question, Option A, Option B, Option C, Option D, Correct Answer (A/B/C/D)

2

Click Bulk Import Button

Green "üì• Bulk Import" button appears next to "Add Question" in both Daily and Competitive sections.

3

Paste CSV Content

Large textarea accepts CSV data. Each line represents one question with comma-separated values.

4

Automatic Processing

System validates format, checks for errors, imports valid questions, and reports success/failure counts.

üìã CSV Format Example
Question,Option A,Option B,Option C,Option D,Correct Answer What is H2O?,Water,Oxygen,Hydrogen,Carbon,A What is CO2?,Water,Carbon Dioxide,Oxygen,Nitrogen,B What is photosynthesis?,Process of making food,Respiration,Digestion,Circulation,A What is mitosis?,Cell division,Cell death,Cell growth,Cell mutation,A What is DNA?,Code of life,Protein,Vitamin,Mineral,A
‚úÖ Validation & Error Handling
  • ‚úì
    Format Validation: Each line must have exactly 6 comma-separated values. Lines with incorrect format are automatically skipped with error count.
  • ‚úì
    Answer Validation: Correct answer must be A, B, C, or D (case-insensitive). Invalid answers result in question being rejected.
  • ‚úì
    Batch Processing: All valid questions are imported even if some fail. System continues processing entire CSV despite individual errors.
  • ‚úì
    Success Reporting: Clear feedback shows "Imported X questions. Y failed." with color-coded success (green) or error (red) messages.
  • ‚úì
    Empty Line Handling: Blank lines are automatically filtered out. No errors thrown for whitespace-only lines.

üîí Security Implementations
  • ‚úì
    Authentication: JWT tokens with 7-day expiration, secure signing with HS256 algorithm, and automatic token refresh capability.
  • ‚úì
    Password Security: bcrypt hashing with 10 salt rounds. Passwords never stored in plain text. Secure comparison algorithms prevent timing attacks.
  • ‚úì
    XSS Protection: Input sanitization on client and server. HTML escaping for all user-generated content. Content Security Policy headers.
  • ‚úì
    CORS Configuration: Controlled cross-origin resource sharing. Whitelist of allowed origins. Credential handling for authenticated requests.
  • ‚úì
    MongoDB Security: Mongoose schema validation prevents injection attacks. IP whitelist on Atlas. Database user with limited permissions.
  • ‚úì
    Environment Variables: All secrets in .env files (not in git). Different configs for dev/prod. Vercel environment variable management.
‚ö° Performance Optimizations
Frontend
Debounced search (300ms), lazy loading, skeleton screens, CSS variables, event delegation, and optimized rendering.
Backend
Connection pooling, cached database connections, indexed queries, lean Mongoose queries, and selective field projection.
Database
Indexes on email, topic names, and dates. Query optimization. Efficient aggregation pipelines for complex queries.
Images
CDN delivery, lazy loading, fallback icons, error handling, and automatic Drive URL conversion for optimal loading.
